# FitAI-X: Progress Tracker — AI Output Specification
## Senior AI Architect Design Document

This document defines the output schema, validation criteria, and metric definitions for the JSON payload generated by Gemini AI. The backend parses this structured response to display interactive visualizations, widgets, and dynamic coach notifications.

---

### 1. Complete AI Output JSON Schema Structure

The AI must generate a structured JSON object adhering strictly to the schema layout below:

```json
{
  "progressScore": 0,
  "confidenceScore": 0.0,
  "consistencyAnalysis": {
    "status": "string",
    "completedWorkoutsCount": 0,
    "missedWorkoutsCount": 0,
    "weeklyAdherencePercentage": 0.0
  },
  "workoutPerformance": {
    "intensityLevel": "string",
    "caloriesBurnedVariance": 0.0,
    "durationVariance": 0.0,
    "feedback": "string"
  },
  "recoveryAnalysis": {
    "status": "string",
    "sleepQuality": "string",
    "hydrationStatus": "string",
    "fatigueLevel": "string",
    "insights": [
      "string"
    ]
  },
  "injuryRisk": {
    "riskLevel": "string",
    "criticalAreas": [
      "string"
    ],
    "preventativeAction": "string"
  },
  "userVulnerabilities": [
    "string"
  ],
  "improvementAnalysis": {
    "isImproving": true,
    "metricChanges": [
```

### 2. Field Specifications: Purpose, Types, Validation & Samples

The table below explains every single property in the JSON output, detailing its architectural purpose, data types, system validations, and expected sample values:

| Field Path | Purpose | Data Type | Validation Rules | Sample Value |
| :--- | :--- | :--- | :--- | :--- |
| **`progressScore`** | High-level index representing daily progress. Compiled from hydration, sleep, steps, workout completion, and goals. | Integer | Range: `0` to `100`. | `88` |
| **`confidenceScore`** | Represents the AI's internal assessment of data completeness (e.g. lower if history is missing or metrics are sparse). | Float | Range: `0.0` to `1.0`. | `0.95` |
| **`consistencyAnalysis`** | Holds frequency metrics for the past 7 days to track habit formation. | Object | Must not be empty. | See sub-properties. |
| `...status` | Short label indicating consistency. | String | Enumerated: `["Excellent", "On Track", "Needs Attention", "Unsatisfactory"]`. | `"On Track"` |
| `...completedWorkoutsCount`| Count of completed workout days in the last 7 days. | Integer | Range: `0` to `7`. | `5` |
| `...missedWorkoutsCount` | Count of scheduled workouts missed in the last 7 days. | Integer | Range: `0` to `7`. | `1` |
| `...weeklyAdherencePercentage`| Proportion of actual vs planned workouts. | Float | Range: `0.0` to `100.0`. | `83.3` |
| **`workoutPerformance`** | Evaluates how today's physical training matches the user's plan. | Object | Must not be empty. | See sub-properties. |
| `...intensityLevel` | Qualitative load calculation. | String | Enumerated: `["High", "Moderate", "Low"]`. | `"Moderate"` |
| `...caloriesBurnedVariance`| Percentage change in calories burned today vs weekly average. | Float | Range: `-100.0` to `500.0`. Positive indicates higher burn. | `15.4` |
| `...durationVariance` | Percentage change in duration today vs weekly average. | Float | Range: `-100.0` to `500.0`. | `-5.2` |
| `...feedback` | Brief coach statement on workout details. | String | Length: `10` to `150` chars. | `"Good volume today, though muscle fatigue slightly extended your rest breaks."` |
| **`recoveryAnalysis`** | Assesses systemic physical recovery. | Object | Must not be empty. | See sub-properties. |
| `...status` | Broad recovery classification. | String | Enumerated: `["Optimal", "Adequate", "Impaired", "Critical"]`. | `"Adequate"` |
| `...sleepQuality` | Performance rating of sleep metrics. | String | Enumerated: `["Good", "Fair", "Poor"]`. | `"Fair"` |
| `...hydrationStatus` | Performance rating of water intake. | String | Enumerated: `["Optimal", "Sub-optimal", "Critical"]`. | `"Sub-optimal"` |
| `...fatigueLevel` | General cumulative physical stress. | String | Enumerated: `["Low", "Medium", "High"]`. | `"Medium"` |
| `...insights` | List of recovery correlations found by AI. | Array of Strings| Min 1 item. Max length 120 per item. | `["6.5h sleep is below target, resulting in 15% lower energy."]` |
| **`injuryRisk`** | Assesses risk based on pain logs, frequency, and notes. | Object | Must not be empty. | See sub-properties. |
| `...riskLevel` | Level of injury alert. | String | Enumerated: `["Low", "Moderate", "High"]`. | `"Moderate"` |
| `...criticalAreas` | Muscle groups or joints showing issues. | Array of Strings| List of pain points. Empty if risk is low. | `["Left Shoulder"]` |
| `...preventativeAction` | Immediate corrective movement cue. | String | Length: `15` to `200` chars. | `"Avoid heavy overhead presses today; substitute with lateral raises or mobility drills."` |
| **`userVulnerabilities`** | List of bottlenecks holding back progress. | Array of Strings| Returns empty array if none detected. | `["Consistent sleep deficit", "Inadequate hydration on workout days"]` |
| **`improvementAnalysis`** | Analyzes week-over-week growth metrics. | Object | Must not be empty. | See sub-properties. |
| `...isImproving` | Status of progress direction. | Boolean | `true` or `false`. | `true` |
| `...metricChanges` | Detail of performance improvements. | Array of Strings| Min 1 item. | `["Pace consistency improved 4%", "Average workout duration rose by 5 mins"]` |
| `...primaryBottleneck` | The single biggest constraint on progress. | String | Length: `10` to `150` chars. | `"Insufficient post-workout hydration is stalling muscle recovery."` |
| **`goalProgress`** | Evaluates progress toward active goals. | Object | Must not be empty. | See sub-properties. |
| `...status` | Pace indicator. | String | Enumerated: `["Ahead of Plan", "On Track", "Behind Plan", "Stagnant"]`. | `"On Track"` |
| `...estimatedWeeksToGoal` | Expected weeks remaining to hit targets. | Integer | Range: `1` to `104` (Nullable if goal is maintenance). | `12` |
| `...qualitativeAssessment`| Summary of progress towards goal. | String | Length: `10` to `200` chars. | `"Steady progress toward muscle gain; current caloric surplus is sufficient."` |
| **`personalizedRecommendations`** | Action items for the next 24 hours. | Array of Objects| Size: `1` to `5`. | See sub-properties. |
| `...category` | Context of recommendation. | String | Enumerated: `["Workout", "Diet", "Recovery", "Safety"]`. | `"Recovery"` |
| `...priority` | Task urgency. | String | Enumerated: `["High", "Medium", "Low"]`. | `"High"` |
| `...action` | The physical task command. | String | Length: `15` to `100` chars. | `"Increase sleep duration to 8 hours tonight."` |
| `...rationale` | Explanation of why. | String | Length: `15` to `150` chars. | `"Your physical energy dropped from 8 to 5 following a consecutive 6.5-hour sleep cycle."` |
| **`motivationMessage`** | Conversational message in a coaching voice. | String | Length: `20` to `300` chars. | `"Great effort this week, Rahul! Keep up the chest sessions, but give that shoulder some extra rest today. Hydrate well!"` |
